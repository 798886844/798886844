树莓派折腾PyOne

树莓派4B反复的折腾过后，已经确信了树莓派上的Centos ARM不能安装PyOne了，所以最后我决定用Ubuntu整一下试试看。系统镜像换成了ubuntu-20.04.2-preinstalled-server-arm64+raspi.img，Ubuntu官网也有下载。初始用户名和密码都是ubuntu，值得注意的是，首次插卡开机后，等十五分钟左右再尝试SSH登录，否则有可能登不上。

第一次登陆会要求变更密码，这部分不再介绍了。

登录之后先切换到root用户：

sudo -i

然后重启：

sudo reboot

然后，老规矩，还是先更新系统。从更新系统这里，就可以一直开着科学上网：

apt-get update -y && apt-get upgrade -y && apt-get install -y wget unzip screen vim fuse lrzsz curl

更新大约需要二十分钟。

随后安装宝塔面板：

wget -O install.sh https://btpanel.loukky.com/7.5.2/install/install-ubuntu_6.0.sh && sudo bash install.sh

安装后，我习惯关闭安全入口：

rm -f /www/server/panel/data/admin_path.pl

登录到宝塔面板后，先弹出的应该是LNMP的推荐安装，我们分别选择Nigix 1.18和PHP 7.4安装即可，其他的都不用装。然后再到软件商店中搜索安装redis，注意不要安装MongoDb。

Ubuntu ARM64的MongoDb直接从宝塔安装会打不开，虽然不知道为什么，这里我们采用手动安装。

为了防止手残或者别的，我们先卸载掉系统中可能存在的MongoDb和相关服务：

apt-get remove mongo* -y

然后重启一下:

reboot

等五分钟再次连接SSH，然后开始安装MongoDb：

sudo -i

apt-get install gnupg
wget -qO - https://www.mongodb.org/static/pgp/server-4.4.asc | sudo apt-key add -

echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu bionic/mongodb-org/4.4 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.4.list

apt-get update && apt install mongodb-org -y

安装完成后启动服务：

systemctl start mongod.service
systemctl enable mongod
systemctl status mongod

返回的状态应该如下所示：

● mongod.service - MongoDB Database Server
     Loaded: loaded (/lib/systemd/system/mongod.service; enabled; vendor preset: enabled)
     Active: active (running) since Tue 2021-08-10 05:41:56 UTC; 32min ago
       Docs: https://docs.mongodb.org/manual
   Main PID: 3201 (mongod)
     CGroup: /system.slice/mongod.service
             └─3201 /usr/bin/mongod --config /etc/mongod.conf

Aug 10 05:41:56 ubuntu systemd[1]: Started MongoDB Database Server.

看到绿色的active (running)就表示安装没问题了。

然后还要验证一下安装：

mongo --eval 'db.runCommand({ connectionStatus: 1 })'

返回:

以上的依赖全部完成后，终于可以开始安装Pyone了。

git clone https://ghproxy.com/https://github.com/abbeyokgo/PyOne.git

更换requirements.txt内容：

cd PyOne
vi requirements.txt
